## 一、项目概述

### 1.1 项目背景

基于2026年小游戏市场调研，路径解谜类游戏（如《箭了又箭》）在畅玩榜表现优异，IAA变现模式成熟。本项目旨在开发一款差异化的箭头解谜游戏，通过外卖配送题材包装，增强传播性和用户粘性。

### 1.2 项目目标

| 目标类型 | 具体目标 |
|----------|----------|
| **商业目标** | IAA变现为主，目标日活1000+，月广告收入5000+ |
| **产品目标** | 完成100关卡，留存率30%+，通关率60%+ |
| **开发目标** | MVP版本4周完成，个人开发，零外包成本 |

### 1.3 目标用户

| 用户画像 | 特征描述 |
|----------|----------|
| **年龄** | 18-35岁，以年轻人为主 |
| **性别** | 男女均衡，女性用户比例可能更高 |
| **场景** | 碎片化时间（通勤、排队、午休） |
| **偏好** | 休闲益智类，不追求高难度 |
| **行为** | 单次5-10分钟，日均启动3-5次 |

## 二、功能需求

### 2.1 核心玩法

#### FR-001：骑手发射机制

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-001 |
| **优先级** | P0（必须） |
| **描述** | 玩家点击骑手后，骑手沿预设方向直线移动 |
| **输入** | 用户点击/触摸骑手 |
| **输出** | 骑手开始移动动画 |
| **规则** | - 骑手只能沿方向直线移动<br>- 移动速度恒定<br>- 移动过程中不可暂停/逆转 |

#### FR-002：碰撞检测机制

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-002 |
| **优先级** | P0（必须） |
| **描述** | 骑手移动过程中检测碰撞，触发相应反馈 |
| **碰撞类型** | - 碰到障碍物 → 延迟/弹回起点<br>- 碰到其他骑手 → 撞车，两者都弹回<br>- 到达出口 → 成功送达，骑手消失 |
| **视觉反馈** | - 撞车：表情变化 + 台词弹出<br>- 成功：开心表情 + 成功台词 |

#### FR-003：关卡胜利判定

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-003 |
| **优先级** | P0（必须） |
| **描述** | 所有骑手成功送达后判定关卡胜利 |
| **胜利条件** | 关卡内所有骑手全部到达出口 |
| **结算内容** | - 显示通关时间<br>- 显示本关评分<br>- 弹出下一关入口 |

#### FR-004：关卡失败判定

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-004 |
| **优先级** | P0（必须） |
| **描述** | 骑手超时或无法完成时判定失败 |
| **失败条件** | - 有骑手超出时限未送达<br>- 或用户主动放弃 |
| **失败处理** | - 弹出失败面板<br>- 提供看广告复活选项<br>- 提供重新开始选项 |

### 2.2 关卡系统

#### FR-005：关卡难度梯度

| 阶段 | 关卡范围 | 骑手数量 | 阻碍类型 | 特殊机制 |
|------|----------|----------|----------|----------|
| **入门** | 1-10 | 2-3个 | 无 | 无 |
| **基础** | 11-30 | 3-4个 | 堵车区 | 无 |
| **进阶** | 31-50 | 4-5个 | 堵车+红绿灯 | 时间限制 |
| **高级** | 51-80 | 5-7个 | 多种阻碍 | VIP/加急单 |
| **专家** | 81-100 | 6-8个 | 全阻碍组合 | 全特殊机制 |

#### FR-006：关卡选择界面

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-006 |
| **优先级** | P0（必须） |
| **描述** | 用户可浏览并选择关卡 |
| **功能** | - 关卡列表展示<br>- 已通关关卡标记<br>- 未解锁关卡显示锁图标<br>- 返回主菜单按钮 |

### 2.3 特殊机制（扩展功能）

#### FR-007：红绿灯机制

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-007 |
| **优先级** | P1（重要） |
| **描述** | 周期性切换的通行控制 |
| **机制** | - 红灯3秒 → 绿灯3秒 → 循环<br>- 红灯时骑手无法通过<br>- 绿灯时骑手正常通行 |
| **视觉** | 红绿灯状态清晰可见 |

#### FR-008：VIP订单机制

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-008 |
| **优先级** | P1（重要） |
| **描述** | 必须优先送达的订单 |
| **机制** | - VIP骑手有皇冠标识<br>- 必须第一个送达<br>- 否则关卡直接失败 |
| **视觉** | 金色边框 + 皇冠图标 |

#### FR-009：加急订单机制

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-009 |
| **优先级** | P1（重要） |
| **描述** | 有时间限制的订单 |
| **机制** | - 加急骑手有时间条<br>- 时间条持续减少<br>- 耗尽则该骑手失败 |
| **视觉** | 闪电图标 + 倒计时条 |

### 2.4 视觉与音效

#### FR-010：骑手表情系统

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-010 |
| **优先级** | P1（重要） |
| **描述** | 骑手根据状态显示不同表情 |
| **表情类型** | - 正常：专注表情<br>- 成功：开心/得意<br>- 撞车：惊讶/悲伤<br>- 超时：焦急/哭脸 |
| **触发时机** | 状态变化时自动切换 |

#### FR-011：台词系统

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-011 |
| **优先级** | P1（重要） |
| **描述** | 骑手状态变化时弹出台词气泡 |
| **台词类型** | - 发射："冲冲冲！"<br>- 成功："准时送达！"<br>- 撞车："哎呀撞了..."<br>- VIP："老板催单！"<br>- 加急："来不及了！" |
| **显示方式** | 气泡在骑手头顶弹出，2秒后消失 |

### 2.5 广告变现（IAA）

#### FR-012：失败复活广告

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-012 |
| **优先级** | P0（必须） |
| **描述** | 关卡失败时提供看广告复活选项 |
| **触发条件** | 关卡失败时弹出 |
| **广告时长** | 15-30秒可选 |
| **复活效果** | 时间重置，骑手回到起点 |
| **用户选择** | 可选择看广告复活或重新开始 |

#### FR-013：奖励领取广告

| 属性 | 说明 |
|------|------|
| **需求ID** | FR-013 |
| **优先级** | P2（可选） |
| **描述** | 通关一定数量后提供看广告领奖励 |
| **触发条件** | 每通关5关 |
| **奖励内容** | 新骑手皮肤/新地图主题 |
| **广告时长** | 15秒 |

## 五、验收标准

### 5.1 MVP版本验收

| 检查项 | 标准 |
|--------|------|
| 核心玩法 | 骑手发射、碰撞检测、胜负判定完整可用 |
| 关卡数量 | 至少20关可玩 |
| 广告变现 | 失败复活广告正常触发 |
| 基础UI | 关卡选择、游戏场景、结算面板完整 |
