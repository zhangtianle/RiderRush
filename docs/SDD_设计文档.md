## 一、游戏设计

### 1.1 游戏名称

**中文**：外卖冲冲冲
**英文**：Rush Delivery

### 1.2 游戏类型

路径规划型解谜游戏 + 外卖配送题材包装

### 1.3 游戏概念

```
一句话描述：
帮助外卖骑手规划出发顺序，避开障碍，准时送达所有订单。

核心循环：
进入关卡 → 观察骑手分布 → 选择骑手 → 骑手移动 → 
碰撞检测 → 继续选择 → 全部送达 → 关卡胜利 → 进入下一关
```

### 1.4 差异化设计

| 维度 | 传统箭头解谜 | 本游戏设计 |
|------|--------------|------------|
| **角色** | 抽象箭头线条 | 外卖骑手（有表情台词） |
| **阻碍** | 静态障碍物 | 堵车/红绿灯/门禁（动态） |
| **机制** | 单一碰撞弹回 | 延迟/VIP优先/加急时限 |
| **视觉** | 极简线条 | 城市地图+角色表情 |
| **传播** | 无 | 搞怪台词适合短视频 |

## 二、核心玩法设计

### 2.1 骑手移动规则

```
移动机制：
├── 骑手只能沿预设方向直线移动
├── 移动速度恒定（约每秒移动1格）
├── 移动过程中不可暂停/逆转
├── 到达出口自动消失（成功）
└── 碰撞障碍物 → 根据障碍类型处理
```

### 2.2 碰撞类型处理

| 碰撞类型 | 处理方式 | 视觉反馈 |
|----------|----------|----------|
| **出口** | 骑手消失，标记成功 | 开心表情 + 成功台词 |
| **普通障碍** | 弹回起点 | 悲伤表情 + 台词 |
| **堵车区** | 进入减速，不弹回 | 焦急表情 |
| **其他骑手** | 两者都弹回起点 | 撞车表情 + 搞怪台词 |
| **红灯** | 无法通过，等待或弹回 | 等待动画 |

### 2.3 关卡难度曲线

```
难度递进设计：

关卡 1-10   [入门]
├── 骑手数：2-3
├── 阻碍：无
├── 机制：无
├── 目的：熟悉基本操作
└── 预计通关时间：30秒内

关卡 11-30  [基础]
├── 骑手数：3-4
├── 阻碍：堵车区（静态）
├── 机制：无
├── 目的：学会规划顺序
└── 预计通关时间：60秒内

关卡 31-50  [进阶]
├── 骑手数：4-5
├── 阻碍：堵车区 + 红绿灯
├── 机制：时间限制（全局）
├── 目的：加入时间压力
└── 预计通关时间：90秒内

关卡 51-80  [高级]
├── 骑手数：5-7
├── 阻碍：全部类型
├── 机制：VIP订单 + 加急订单
├── 目的：多维度挑战
└── 预计通关时间：120秒内

关卡 81-100 [专家]
├── 骑手数：6-8
├── 阻碍：复杂组合
├── 机制：全部特殊机制
├── 目的：极限挑战
└── 预计通关时间：150秒+
```

## 三、特殊机制设计

### 3.1 红绿灯机制

```
状态周期：
┌─────────────────────────────────────┐
│  红灯(3s) → 绿灯(3s) → 红灯(3s) ... │
└─────────────────────────────────────┘

骑手行为：
├── 绿灯时：正常通过
├── 红灯时：
│   ├── 骑手在红灯前停下等待
│   ├── 等待期间表情变化
│   └── 或可选择弹回重新规划
└── 视觉：红绿灯状态清晰可见
```

### 3.2 VIP订单机制

```
VIP骑手特性：
├── 标识：金色边框 + 皇冠图标
├── 规则：必须第一个送达
├── 失败条件：
│   ├── 其他骑手先送达 → 关卡失败
│   └── VIP骑手超时 → 关卡失败
└── 台词："老板催单了！"

设计目的：
增加策略深度，用户必须优先规划VIP路线
```

### 3.3 加急订单机制

```
加急骑手特性：
├── 标识：闪电图标 + 倒计时条
├── 规则：必须在时限内送达
├── 时间条：持续减少，视觉可见
├── 失败条件：时间耗尽 → 该骑手失败
└── 台词："来不及了！"

时间设置：
├── 入门关卡：加急时限60秒
├── 进阶关卡：加急时限30秒
└── 专家关卡：加急时限15秒
```

### 3.4 电梯机制（可选扩展）

```
电梯特性：
├── 可同时承载2-3个骑手
├── 骑手进入后等待其他骑手
├── 同时发射 → 更高效通关
└── 策略：规划一起上电梯的骑手组
```

## 四、视觉设计

### 4.1 整体风格

| 维度 | 设计方向 |
|------|----------|
| **画风** | Q版简笔画 + 搞怪表情 |
| **色调** | 暖色调为主，生活感 |
| **透视** | 俯视图（城市地图） |
| **对比度** | 高对比度，元素清晰 |

### 4.2 骑手设计

```
骑手造型：
├── 体型：Q版小人，比例约1:2
├── 服饰：外卖制服（黄色/蓝色）
├── 头部：戴头盔，表情丰富
├── 骑乘：电动车简化造型
└── 方向：箭头指示出发方向

表情设计：
├── 正常状态：专注/中性表情
├── 成功送达：开心/得意/挥手
├── 撞车失败：惊讶/悲伤/捂脸
├── 等待红灯：焦急/叹气
├── VIP状态：紧张/看表
└── 加急状态：着急/喊叫

表情切换：
├── 状态变化 → 表情切换
├── 切换动画：0.3秒过渡
└── 保持时间：持续到状态变化
```

### 4.3 阻碍设计

| 阻碍类型 | 视觉表现 |
|----------|----------|
| **堵车区** | 几辆小车堆在一起，红色区域标记 |
| **红绿灯** | 经典红绿灯造型，状态颜色清晰 |
| **门禁杆** | 小区门口升降杆，可升降动画 |
| **墙壁** | 建筑物边缘，灰色实心块 |

### 4.4 地图背景

```
地图设计：
├── 基础地图：城市街道俯视图
├── 区域划分：
│   ├── 街道区域（灰色）
│   ├── 小区区域（绿色）
│   ├── 商业区（蓝色）
│   └── 特殊区域（橙色）
├── 道路网格：清晰可见
└── 装饰元素：树木、建筑简化图
```

### 4.5 UI设计

| UI元素 | 设计 |
|--------|------|
| **关卡选择** | 卡片列表，通关标记，锁定图标 |
| **游戏场景** | 地图居中，骑手分布，时间条 |
| **结算面板** | 通关时间，评分星星，按钮 |
| **失败面板** | 失败提示，复活按钮，重试按钮 |

## 六、关卡设计示例

### 6.1 关卡1（入门）

```
关卡布局：
┌─────────────────────────┐
│                         │
│   [骑手A] → → → 出口1   │
│                         │
│   [骑手B] → → → 出口2   │
│                         │
└─────────────────────────┘

设计说明：
├── 2个骑手，直线路径
├── 无阻碍
├── 任意顺序都可通关
└── 目的：熟悉操作
```

### 6.2 关卡15（基础）

```
关卡布局：
┌─────────────────────────┐
│                         │
│   [骑手A] → → [堵车] → 出口1
│              ↓          │
│   [骑手B] ↓   [骑手C] → 出口2
│           ↓             │
│         出口3           │
│                         │
└─────────────────────────┘

设计说明：
├── 3个骑手
├── 1个堵车区
├── 需要规划顺序避免撞车
└── 正确顺序：C → B → A 或 B → A → C
```

### 6.3 关卡35（进阶）

```
关卡布局：
┌─────────────────────────┐
│  [VIP骑手] → [红灯] → 出口1
│              ↓          │
│   [加急骑手] ↓ [普通骑手] → 出口2
│           ↓             │
│         出口3           │
│  时间限制：90秒         │
└─────────────────────────┘

设计说明：
├── 3个骑手：VIP + 加急 + 普通
├── 红绿灯周期切换
├── VIP必须先送达
├── 加急有时间条
└── 多维度策略挑战
```

## 七、变现设计

### 7.1 IAA变现策略

```
广告触发设计：

主要收入来源：
├── 失败复活广告
│   ├── 触发频率：每次失败可选
│   ├── 广告时长：15-30秒
│   ├── 用户选择：可选，不强制
│   └── 预估触发率：30-40%

辅助收入来源：
├── 奖励领取广告
│   ├── 触发频率：每通关5关
│   ├── 奖励：皮肤/主题
│   ├── 广告时长：15秒
│   └── 预估触发率：20-30%

收入预估（日活1000）：
├── 每用户日均广告展示：3-5次
├── 广告单价（预估）：0.05-0.1元/次
├── 日收入：150-500元
└── 月收入：4500-15000元
```

### 7.2 广告体验优化

```
优化原则：
├── 广告可选触发，不强制
├── 避免开屏广告（用户流失）
├── 避免通关强制广告（负面体验）
├── 频率控制：每5分钟不超过3次
└── 用户可选择跳过（等待15秒）
```
