## Context

游戏"外卖冲冲冲"是一个像素风益智游戏，玩家为外卖骑手规划路线。当前渲染基于 Canvas 2D，GameRenderer.ts 负责所有绘制。UI 层使用 HTML/CSS 覆盖在 Canvas 上。已有像素精灵系统（drawPixelSprite）、像素边框系统（drawPixelBorder）、粒子系统等基础设施。

当前视觉问题：网格是简单双色棋盘格、HUD 是纯色方块、菜单缺少氛围、特效偏重。

## Goals / Non-Goals

**Goals:**
- 游戏网格从抽象棋盘格升级为有街道感的静态像素纹理
- HUD 元素使用像素风边框和小图标，提升组件感
- 主菜单增加氛围元素（标题框、城市剪影）
- 特效反馈调低强度，保持克制
- 色彩体系新增两个点缀色，丰富视觉层次

**Non-Goals:**
- 不做动态天气、粒子雨雪等动画效果
- 不做视差滚动、背景动画
- 不改变游戏玩法逻辑
- 不改变现有像素精灵（骑手、出口等）
- 不新增外部依赖

## Decisions

### 1. 网格纹理：静态像素纹理 vs 动态渲染

**选择**：在 renderGrid() 中用静态像素纹理替代纯色填充。

每个格子根据位置计算一个伪随机纹理类型（3-4种变化），绘制微妙的明暗像素点阵模拟沥青质感。路沿石在网格边缘绘制，车道虚线在特定行/列绘制。建筑剪影在网格外围绘制。

**理由**：静态纹理性能好，不影响游戏流畅度，且能营造街道氛围。

**替代方案**：预渲染纹理贴图到 offscreen canvas 再 blit — 性能更好但增加复杂度，当前网格尺寸不大，直接绘制足够。

### 2. HUD 实现：Canvas 绘制 vs HTML/CSS

**选择**：保持 HTML/CSS 实现 HUD，用 CSS 像素风边框和伪元素图标。

**理由**：HUD 已经是 HTML 实现，改动成本低。CSS 的 box-shadow 可以模拟像素双层边框。用 CSS 伪元素或 inline SVG 做小图标。

**替代方案**：把 HUD 移到 Canvas 内绘制 — 统一性好但需要处理文字渲染和响应式，成本太高。

### 3. 城市剪影：CSS 实现 vs Canvas

**选择**：用 CSS 在菜单底部绘制静态像素城市剪影。

用 CSS clip-path 或 box-shadow 拼出建筑轮廓，或者用一个重复的像素图案 background。

**理由**：菜单是 HTML 层，用 CSS 实现最简单。可以用 CSS 渐变 + clip-path 模拟像素建筑。

### 4. 特效参数调整方式

**选择**：在 GameConstants.ts 中集中定义特效参数常量，GameRenderer.ts 引用这些常量。

**理由**：便于后续微调，不需要到处找魔法数字。

### 5. 色彩体系

**选择**：新增两个点缀色到 COLORS 常量中，每个颜色只在特定场景使用：
- #4ECDC4（青绿）：送达成功光圈、出口高亮
- #FF6B9D（柔粉）：VIP 相关、特殊状态

**理由**：两个新色足够丰富视觉层次，不会让配色变杂乱。

## Risks / Trade-offs

- **性能**：网格纹理绘制比纯色填充略慢 → 当前网格尺寸（8x6）很小，影响可忽略。如果后续关卡网格变大，可改为 offscreen canvas 缓存纹理。
- **CSS 兼容性**：像素风边框用 box-shadow 实现，低版本浏览器可能不支持多层 shadow → 目标是现代浏览器和微信 WebView，基本无问题。
- **视觉一致性**：新旧元素混搭可能风格不统一 → 所有新增元素都遵循像素风原则：直角、无抗锯齿、有限色板。
